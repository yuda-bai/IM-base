## 二、根因分析

### 核心问题链

```
服务器重启
  → MySQL / Redis / BT Panel 均未设置开机自启
  → Go 后端启动时 panic（连不上 MySQL 和 Redis）
  → nginx 虽然正常运行，但后端端口不匹配（8081 vs 8080）
  → 手动启动后发现数据库配置错误（ginchat_user 不存在）
  → 修复后进程守护管理器反复重启导致 WebSocket 频繁断连
  → WebSocket 不稳定 → 消息丢失
```

### 关键教训

1. **服务自启**：MySQL、Redis、Go 后端都设了开机自启才能避免服务器重启后需要手动恢复
2. **配置文件一致性**：本地 `config/app.yml` 用 `root:123456`，服务器上被覆盖成 `ginchat_user:Ginchat@12345`，而这个 MySQL 用户不存在
3. **WebSocket 不可靠**：通过 nginx 代理的 WebSocket 受网络波动影响大，消息发送不能只依赖 WS，必须有 HTTP 备用通道
4. **符号链接不完整**：版本管理的 releases/ 目录建了但 current、frontend/dist 等符号链接没建全，导致 nginx 找不到前端文件

## 三、修复内容详情

### 3.1 服务启动修复

| 服务 | 问题 | 修复命令 |
|------|------|----------|
| MySQL | 未运行 | `sudo /etc/init.d/mysqld start` |
| Redis | 未运行 | `sudo /etc/init.d/redis start` |
| BT Panel | 未运行 | `sudo bt` → 选 1 重启 |
| Go 后端 | 未运行 | `./ginchat_server &` |

### 3.2 配置修复

| 文件 | 问题 | 修复 |
|------|------|------|
| `/www/wwwroot/ginchat-server/config/app.yml` | mysql.dns 用 `ginchat_user` 但不存在 | 改为 `root:123456` |
| `/www/server/panel/vhost/nginx/YOUR_SERVER_IP.conf` | proxy_pass 8081 | 改为 8080 |
| supervisor ginchat.ini | directory 不对 | 设为 `/www/wwwroot/ginchat-server` |

### 3.3 版本管理目录结构

建立 `releases/<版本号>/` + `current` 符号链接体系：

```
/www/wwwroot/ginchat-server/
├── releases/
│   ├── initial/
│   ├── 20260714-v1.01/
│   ├── 20260714-v1.02/
│   └── 20260714-v1.03/          ← 最新版本
├── current → releases/20260714-v1.03/
├── ginchat_server → current/ginchat_server
└── frontend/dist → ../current/frontend/dist/
```

### 3.4 WebSocket 稳定性优化

| 改动 | 旧值 | 新值 | 文件 |
|------|------|------|------|
| 心跳间隔 | 30s | 15s | models/message.go |
| 读超时 | 90s | 45s | models/message.go |
| 心跳失败处理 | 静默退出 | 主动 close node | models/message.go |

### 3.5 HTTP 发送消息备用通道（本次核心修复）

新增 `POST /user/SendMessageHttp` 接口：
- 消息通过 HTTP POST 可靠写入数据库
- 再通过 WebSocket 推送给在线用户
- 前端 `handleSendMessage` 改为 HTTP 优先，WS 备用

**修改文件：**
- `models/message.go` — 新增 `SendMessageViaHTTP()` 函数
- `service/user_service.go` — 新增 `SendMessageHttp()` handler
- `router/router.go` — 注册路由 `POST /user/SendMessageHttp`
- `frontend/src/api/user.js` — 新增 `sendMessageHttp()` API 调用
- `frontend/src/views/ChatRoom.vue` — 发送逻辑改为 HTTP POST

### 3.6 数据库修复

```sql
ALTER TABLE ginchat.messages ADD COLUMN is_read TINYINT(1) DEFAULT 0 AFTER type;
```
